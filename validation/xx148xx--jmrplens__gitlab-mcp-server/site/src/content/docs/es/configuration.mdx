## Opciones principales

| Variable                 | Predeterminado       | DescripciĂłn                                                                                                                                                                                                    |
| ------------------------ | -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

Esto no elimina los schemas del descubrimiento dinĂˇmico: usa `gitlab_find_action` para recuperar schemas exactos antes de llamar a `gitlab_execute_action`, o lee `gitlab://tools/{id}` para una entrada del manifiesto. ## Orden de carga de la configuraciĂłn

El servidor carga la configuraciĂłn en el siguiente orden (las fuentes posteriores anulan las anteriores):

1. **`~/.gitlab-mcp-server.env`** â€” Valores predeterminados a nivel de usuario (directorio home)
2. **`.env`** â€” ConfiguraciĂłn a nivel de proyecto (directorio de trabajo actual)
3. **Variables de entorno del sistema** â€” Variables exportadas en el shell
4. **Flags de CLI** â€” Argumentos de lĂ­nea de comandos (prioridad mĂˇs alta)

:::note
En modo HTTP, `GITLAB_TOKEN` no se carga desde variables de entorno. Cada cliente proporciona su propio token mediante la peticiĂłn HTTP. Los clientes pueden enviar una cabecera `GITLAB-URL` solo cuando el servidor arranca sin `--gitlab-url`.

Cuando `--gitlab-url` estĂˇ configurado, es autoritativo y los valores `GITLAB-URL` enviados por clientes se ignoran y se registran. Otras opciones de polĂ­tica del servidor, incluido el tamaĂ±o de esquema (`--meta-param-schema`) y los lĂ­mites de tasa (`--rate-limit-rps`, `--rate-limit-burst`), no se pueden sobreescribir por peticiĂłn.
:::

## Modo solo lectura

Habilita `GITLAB_READ_ONLY=true` para restringir el servidor a operaciones de solo lectura. Todas las herramientas que crean, actualizan o eliminan recursos se desactivan. Esto es Ăştil para:

- Entornos de auditorĂ­a y cumplimiento
- Servidores compartidos donde los usuarios solo deben consultar datos
- Tokens con scope `read_api`

## Modo seguro

- Revisar operaciones antes de ejecutarlas (dry-run)
- Entornos de formaciĂłn donde se quiere ver el comportamiento de las herramientas
- Depurar parĂˇmetros de herramientas sin efectos secundarios

:::note
Si `GITLAB_READ_ONLY=true` tambiĂ©n estĂˇ configurado, tiene precedencia â€” las herramientas de escritura se desactivan completamente en lugar de mostrar una vista previa.
:::
